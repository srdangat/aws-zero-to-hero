# Day 1: Fundamentals of AWS and Cloud Computing

## 1. AWS Pricing Models

* **Pay-as-you-go (On-Demand):** Pay only for what you use by the second/hour. No upfront commitments.
  * **Best for:** Startups, learners, and unpredictable workloads.

* **Reserved Instances:** Commit to 1 or 3 years for a significant discount (up to 72%).
  * **Best for:** Steady-state, predictable enterprise workloads.

* **Spot Instances:** Bid on unused AWS capacity at massive discounts (up to 90%). Instances can be interrupted with short notice.
  * **Best for:** Fault-tolerant, flexible, and batch processing workloads.

## 2. Cloud Deployment Models

* **On-premises (Private Cloud):** Infrastructure is kept locally. You own and manage the hardware.
  * **Best for:** Government agencies, high-security data, and legacy systems.

* **Public Cloud:** Infrastructure is owned by a third-party provider (like AWS) and accessed over the internet.
  * **Best for:** Startups, modern web applications, and fast-scaling businesses.

* **Hybrid Cloud:** A mix of on-premises and public cloud, allowing data to move between them.
  * **Best for:** Enterprises transitioning to the cloud, or those needing to burst to the public cloud during high traffic while keeping core data secure locally.

## 3. Cloud Service Models

* **IaaS (Infrastructure as a Service):** Rent IT infrastructure (servers, VMs, storage, networks). 
  * **Examples:** AWS EC2, Google Compute Engine, Microsoft Azure VMs.

* **PaaS (Platform as a Service):** Provides a framework for developers to build/deploy apps without managing underlying hardware. 
  * **Examples:** AWS Elastic Beanstalk, AWS Lambda (Serverless/Function-as-a-Service), Heroku.

* **SaaS (Software as a Service):** Finished applications delivered over the internet directly to end-users. 
  * **Examples:** AWS WorkMail, Google Workspace, Zoom, Salesforce.

* **2014:** The Revolution (Lambda/Serverless)
* **2018:** The Silicon (Graviton custom chips)
* **2023:** The AI Pivot (Amazon Bedrock)
